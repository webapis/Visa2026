## Word letter (`App_Inv_And_WP_Letter.docx`)

Generated by `tools/GenerateTemplates` → `MakeAppInvAndWPLetterTemplate`. Merge model prefix `ds` matches `AppInvAndWPLetterReportDef` (same logical fields as the XAF report, sourced from **`Application`** and related `ProjectContract`, `Company`, `CompanyHead`, etc.). **Page margins:** **left and right both 1200** twips (~2.12 cm) — symmetric, narrower than default L1–L3 (1800 / 1800). **Vertical rhythm:** tighter `w:spacing` between header lines, before/after salutation, between justified body paragraphs (`w:after` **100** twips on description + request). Responsibility paragraph uses **`w:after` 40** twips and **no empty paragraph** before the Goşundy lines so attachments sit closer to the clause above. Before signatory: blank para **`w:after` 80**; signatory cells **`w:before` 320** twips — see `FormalCompanyLetterLayout` (`InvAndWP*` constants).

| `ds.*` key | `Application` / graph |
|------------|------------------------|
| `FullApplicationNumber`, `ApplicationDate` | `Application` |
| `ProjectContract_Ministry_RecipientBlock` | `ProjectContract.Ministry.RecipientBlock` — **authoritative ministry addressee text from the database** (full string; still passed for parity / other consumers). |
| `ProjectContract_Ministry_RecipientBlock_Line1`, `ProjectContract_Ministry_RecipientBlock_Line2`, `ProjectContract_Ministry_RecipientBlock_HasLine2` | **Derived at merge time only** in `AppInvAndWPLetterReportDef` / `AppVisaAndWPExtLetterReportDef` via `MinistryRecipientBlockFormatter.SplitIntoAddressLines` — **no static Türkmenenergo wording in the `.docx`**. Word layout: **borderless table** (narrow **spacer** cell + **remaining printable width** for the address column); both lines **`w:jc left`** in that column so they share one left edge (block on the right). DocxTemplater `{?{…}}…{{/}}` omits line 2 text when `HasLine2` is false. |
| `Urgency_NameTm` | `Urgency.NameTm` |
| `ProjectContract_Ministry_FormOfAddress` | `ProjectContract.Ministry.FormOfAddress` |

**Placeholder parity:** Body sentences use the same merge keys as the map / `AppInvAndWPReport` (`[Company.Name]` → `Company_Name`, `[TotalPersonCount]` / `[TotalPersonCountText]`, `[VisaPeriod_NameTm]`, `[VisaCategory_NameTm]`).

**Goşundy (attachments) — Word vs XAF**

| Channel | Text |
|--------|------|
| **Word** (`App_Inv_And_WP_Letter.docx`) | `1. {{TotalPersonCount}}-pasport kopiýalary,` and `2. Goşundy ({{TotalPersonCount}}-daşary ýurt raýatynyň maglumaty)` (two paragraphs; same `Application.TotalPersonCount` as the request paragraph). |
| **XAF** (`xrLabelAttachments`) | Unchanged: sanawy + maglumat expression with `[TotalPersonCount]` / `[TotalPersonCountText]`. |

**Word typography vs XAF RTF (`App_Inv_And_WP_app.jpg`):**

| Region | Word template (scan) | XAF `xrRichBody2` (map `**…**`) |
|--------|------------------------|----------------------------------|
| `ProjectContract_Description` (opening body) | Justified, plain, content only (no "Maksady:" label), **first-line indent 720 twips** — same `w:pPr` as paras 2–3 (no extra leading spaces in static text) | `xrRichBody1`: `[ProjectContract_Description]` only, justified `\fi720` |
| `[Company.Name]` in request paragraph | **Bold** | Plain in map excerpt |
| `[TotalPersonCount] ([TotalPersonCountText])`, `[VisaPeriod_NameTm] möhlet`, `[VisaCategory_NameTm] çakylyk we iş rugsatnamasyny` | Plain | **Bold** in map |
| Ministry recipient (addressee) block | **Two lines** when `RecipientBlock` splits (see formatter). Word: **table** — spacer cell + address cell (**left-aligned** lines, shared left edge; block on the right). Single logical line if no split. Urgency follows **after** the table (left). | Single `xrLabelRecipient` text block |

Salutation: **bold, centered, no underline** (Word standard; scan sample is underlined). Urgency: **italic, black, no underline** (Word standard; scan is red + underlined); in Word template it appears **after** the addressee, not between date and addressee. XAF remains italic only. Attachments: Word uses scan wording (see **Goşundy** table above); XAF uses `xrLabelAttachments` expression.
